## Overview

The Workflows section provides visibility into all CI/CD pipelines that publish to or consume from your Fly Registry.

## Workflow Details

Click on any workflow run to view details:

<Tabs>
  <Tab title="Summary">
    ### Run Summary

    - **Workflow**: Name and file path
    - **Repository**: Source repository link

    - **Commit**: Associated commit hash
    - **Triggered by**: User or event

    │
    ├── Publish

## Filtering Workflows

|--------|-------------|

### Quick Filters

- **My workflows** - Workflows triggered by you
- **Failed today** - Today's failed runs
- **Publishing** - Workflows that publish artifacts
- **Active now** - Currently running

## Workflow Actions

### Cancel Workflow

Stop a running workflow:

<Steps>
  <Step title="Find Workflow">
    Locate the running workflow
  </Step>
  
  <Step title="Click Cancel">
    Click the **Cancel** button
  </Step>
  
  <Step title="Confirm">
    Confirm the cancellation
  </Step>
</Steps>

### Re-run Workflow

Re-run a completed workflow:

```
Re-run options:
├── Re-run all jobs
├── Re-run failed jobs only
└── Re-run from specific job
```

### Download Logs

Export workflow logs:

```bash
# Via Fly CLI
fly workflow logs <workflow-id> --output logs.txt
```

## Troubleshooting Workflows

<AccordionGroup>
  <Accordion title="Workflow Failed" icon="circle-xmark" defaultOpen>
    ### Investigating Failures
    
    <Steps>
      <Step title="Check Logs">
        View the job logs to identify the failure point
      </Step>
      
      <Step title="Identify Error">
        Look for error messages and exit codes
      </Step>
      
      <Step title="Check Common Issues">
        - Authentication failures
        - Network timeouts
        - Missing dependencies
        - Resource limits
      </Step>
      
      <Step title="Fix and Re-run">
        Address the issue and re-run the workflow
      </Step>
    </Steps>
  </Accordion>
  
  <Accordion title="OIDC Authentication Failed" icon="key">
    **Error:** `Failed to authenticate with OIDC`
    
    **Possible causes:**
    - Trust policy doesn't match workflow
    - Repository not configured for OIDC
    - Token request failed
    
    **Solutions:**
    1. Check trust policy in repository settings
    2. Verify `id-token: write` permission in workflow
    3. Review OIDC configuration in Fly Web
  </Accordion>
  
  <Accordion title="Publish Failed" icon="upload">
    **Error:** `Failed to publish artifact`
    
    **Possible causes:**
    - Version already exists
    - Insufficient permissions
    - Network timeout
    
    **Solutions:**
    1. Check if version exists already
    2. Verify publish permissions
    3. Increase timeout and retry
  </Accordion>
  
  <Accordion title="Workflow Not Triggered" icon="play">
    **Symptoms:** Expected workflow didn't run
    
    **Possible causes:**
    - Webhook not configured
    - Branch filter doesn't match
    - Workflow file error
    
    **Solutions:**
    1. Check webhook delivery in Git provider
    2. Review workflow trigger conditions
    3. Validate workflow YAML syntax
  </Accordion>
</AccordionGroup>
