## Configuration Overview

The Fly Desktop App can be configured through:

1. **GUI Settings** - Via the system tray menu
2. **CLI Commands** - Using `fly config`
3. **Configuration File** - Edit `~/.fly/config.yaml` directly

---

## GUI Settings

Access settings through the system tray:

1. Right-click the Fly tray icon
2. Select **Settings**
3. Navigate through the settings panels

### Settings Panels

<Tabs>
  <Tab title="General">
    - **Launch at startup** - Start Fly when your computer boots
    - **Show in menu bar** - Display status in menu bar (macOS)
    - **Theme** - Light, Dark, or System
    - **Language** - Interface language
  </Tab>
  
  <Tab title="Connection">
    - **Registry URL** - Your Fly Registry URL
    - **Proxy settings** - HTTP/HTTPS proxy configuration
    - **Timeout** - Connection timeout in seconds
    - **Retry attempts** - Number of retries on failure
  </Tab>
  
  <Tab title="Package Managers">
    - **Auto-detect** - Automatically detect installed package managers
    - **Auto-configure** - Automatically configure detected package managers
    - **Package manager toggles** - Enable/disable specific package managers
  </Tab>
  
  <Tab title="Sync">
    - **Sync interval** - How often to sync (in minutes)
    - **Sync on startup** - Sync when app starts
    - **Background sync** - Enable background synchronization
  </Tab>
  
  <Tab title="Notifications">
    - **Enable notifications** - Show system notifications
    - **New releases** - Notify on team releases
    - **Sync errors** - Notify on sync failures
    - **Updates** - Notify when app updates are available
  </Tab>
</Tabs>

## CLI Configuration

### View Current Configuration

```bash
# Show all configuration
fly config show

# Show specific setting
fly config get sync.interval
```

### Set Configuration Values

```bash
# Set a value
fly config set sync.interval 30

# Set nested values
fly config set proxy.http "http://proxy.company.com:8080"

# Reset to default
fly config reset sync.interval
```

### Common Configuration Commands

```bash
# Configure all detected package managers
fly config auto

# Configure specific package manager
fly config npm
fly config pip
fly config maven
fly config docker

# Clear all configurations
fly config clear
```

## Configuration File

The main configuration file is located at `~/.fly/config.yaml`:

```yaml
# ~/.fly/config.yaml

# Version of the config file format
version: 1

# Your Fly Registry connection
registry:
  url: https://myteam.jfrog.io
  
# Authentication (managed automatically)
auth:
  token_path: ~/.fly/credentials

# Package manager settings
package_managers:
  npm:
    enabled: true
    scope: "@myteam"
  pip:
    enabled: true
    index_url: auto
  maven:
    enabled: true
    server_id: fly-central
  docker:
    enabled: true
    mirror: true
  go:
    enabled: true
    proxy: true

# Sync settings
sync:
  interval: 1800  # 30 minutes
  on_startup: true
  background: true

# Cache settings
cache:
  enabled: true
  max_size: 5GB
  ttl: 86400  # 24 hours

# Proxy settings
proxy:
  http: null
  https: null
  no_proxy: 
    - localhost
    - 127.0.0.1

# Network settings
network:
  timeout: 60
  retry_count: 3
  retry_delay: 5

# Logging
logging:
  level: info  # debug, info, warn, error
  file: ~/.fly/logs/fly.log
  max_size: 10MB
  max_files: 5

# Notifications
notifications:
  enabled: true
  releases: true
  errors: true
  updates: true

# IDE Integration
ide:
  mcp_server:
    enabled: true
    port: 9222
  
# Auto-update
updates:
  auto_check: true
  auto_install: false
```

## Package Manager Configuration

### npm Configuration

Location: `~/.npmrc` (global) or `.npmrc` (project)

```ini
# Generated by Fly Desktop App
registry=https://myteam.jfrog.io/artifactory/api/npm/npm/
//myteam.jfrog.io/artifactory/api/npm/npm/:_authToken=${FLY_NPM_TOKEN}
always-auth=true
```

### pip Configuration

Location: `~/.pip/pip.conf` (macOS/Linux) or `%APPDATA%\pip\pip.ini` (Windows)

```ini
# Generated by Fly Desktop App
[global]
index-url = https://myteam.jfrog.io/artifactory/api/pypi/pypi/simple
trusted-host = myteam.jfrog.io
extra-index-url = https://pypi.org/simple/
```

### Maven Configuration

Location: `~/.m2/settings.xml`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!-- Generated by Fly Desktop App -->
<settings>
  <servers>
    <server>
      <id>fly-central</id>
      <username>${env.FLY_USER}</username>
      <password>${env.FLY_TOKEN}</password>
    </server>
  </servers>
  
  <mirrors>
    <mirror>
      <id>fly-central</id>
      <mirrorOf>central</mirrorOf>
      <url>https://myteam.jfrog.io/artifactory/maven-remote</url>
    </mirror>
  </mirrors>
</settings>
```

### Docker Configuration

Location: `~/.docker/config.json`

```json
{
  "credHelpers": {
    "myteam.jfrog.io": "fly"
  }
}
```

## Environment Variables

Fly respects the following environment variables:

| Variable | Description |
|----------|-------------|
| `FLY_URL` | Override registry URL |
| `FLY_TOKEN` | Authentication token |
| `FLY_USER` | Username for basic auth |
| `FLY_CONFIG` | Path to config file |
| `FLY_LOG_LEVEL` | Logging level |
| `FLY_NO_AUTO_UPDATE` | Disable auto-updates |
| `FLY_OFFLINE` | Enable offline mode |

### Setting Environment Variables

<Tabs>
  <Tab title="macOS/Linux">
    ```bash
    # In ~/.zshrc or ~/.bashrc
    export FLY_URL="https://myteam.jfrog.io"
    export FLY_LOG_LEVEL="debug"
    ```
  </Tab>
  
  <Tab title="Windows">
    ```powershell
    # In PowerShell profile
    $env:FLY_URL = "https://myteam.jfrog.io"
    $env:FLY_LOG_LEVEL = "debug"
    ```
  </Tab>
</Tabs>

## Project-Level Configuration

Create a `.fly/config.yaml` in your project root for project-specific settings:

```yaml
# .fly/config.yaml - Project level configuration
version: 1

# Override registry for this project
registry:
  url: https://project-specific.jfrog.io

# Publishing configuration
publishing:
  package_type: npm
  auto_version: true
  include_git_info: true
  
# Pre/post hooks
hooks:
  pre_publish:
    - npm test
    - npm run lint
    - npm run build
  post_publish:
    - echo "Published successfully!"

# Files to exclude from publish
ignore:
  - "*.test.js"
  - "*.spec.ts"
  - "__tests__/**"
  - "docs/**"
  - ".env*"
  - "*.md"
```

## Troubleshooting Configuration

<AccordionGroup>
  <Accordion title="Configuration Not Applied" icon="xmark">
    **Symptoms:** Changes to config file not taking effect
    
    **Solutions:**
    1. Restart the Desktop App
    2. Check config file syntax: `fly config validate`
    3. Check file permissions on `~/.fly/config.yaml`
  </Accordion>
  
  <Accordion title="Package Manager Not Configured" icon="box-open">
    **Symptoms:** Package manager still using default registry
    
    **Solutions:**
    1. Run `fly config <package-manager>` to reconfigure
    2. Check package manager config file locations
    3. Ensure environment variables are set
  </Accordion>
  
  <Accordion title="Proxy Issues" icon="globe">
    **Symptoms:** Connection timeouts behind corporate proxy
    
    **Solutions:**
    1. Configure proxy in settings
    2. Add `no_proxy` exceptions for internal hosts
    3. Check proxy authentication requirements
  </Accordion>
</AccordionGroup>
