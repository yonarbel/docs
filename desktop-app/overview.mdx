## Overview

The Fly Desktop App is your local companion that bridges your development environment with JFrog Fly. It runs in the background, managing package manager configurations, providing CLI access, and enabling AI-powered workflows in your IDE.

## Quick Actions

### From System Tray

| Action | Description |
|--------|-------------|
| **Sync** | Refresh package manager configurations |
| **Publish** | Publish current project's artifacts |
| **Search** | Find artifacts in your registry |
| **Logs** | View application logs |
| **Settings** | Open configuration panel |

### From CLI

```bash
# Check status
fly status

# Sync configurations
fly sync

# Quick publish
fly publish

# Search artifacts
fly search "auth"
```

## Version Information

Check your Desktop App version:

```bash
fly version

# Output:
# Fly Desktop App v1.5.0
# CLI: v1.5.0
# MCP Server: v1.2.0
# OS: darwin arm64
```

### Auto Updates

The Desktop App automatically checks for updates. Configure update behavior in Settings:

- **Automatic** - Install updates automatically (recommended)
- **Notify** - Notify when updates are available
- **Manual** - Check for updates manually

---

## Resource Usage

The Desktop App is designed to be lightweight:

| Resource | Typical Usage |
|----------|---------------|
| Memory | 50-100 MB |
| CPU | Less than 1% idle |
| Network | Minimal (only during sync) |
| Disk | 100 MB + cache |

### Reducing Resource Usage

```bash
# Reduce sync frequency
fly config set sync.interval 3600  # Every hour

# Limit cache size
fly config set cache.max_size 1GB

# Disable notifications
fly config set notifications.enabled false
```
