## A

<AccordionGroup>
  <Accordion title="Access Token">
    A credential used to authenticate with the Fly Registry programmatically. Tokens can be personal (tied to a user) or service tokens (for CI/CD).
  </Accordion>
  
  <Accordion title="Agentic Workflow">
    AI-powered workflows that allow developers to manage artifacts using natural language commands in their IDE or CLI.
  </Accordion>
  
  <Accordion title="Artifact">
    A packaged piece of software (npm package, Docker image, JAR file, etc.) stored in the Fly Registry.
  </Accordion>
</AccordionGroup>

## C

<AccordionGroup>
  <Accordion title="CLI (Command Line Interface)">
    The `fly` command-line tool that provides access to all Fly features from your terminal.
  </Accordion>
</AccordionGroup>

## D

<AccordionGroup>
  <Accordion title="Desktop App">
    The Fly Desktop App that runs on your local machine, managing package manager configurations and enabling IDE integration.
  </Accordion>
</AccordionGroup>

## F

<AccordionGroup>
  <Accordion title="Fly Chat">
    AI-powered assistant in Fly Web for searching artifacts and getting help using natural language.
  </Accordion>
  
  <Accordion title="Fly Registry">
    The central repository where your team's artifacts are stored. Accessed via `<your-subdomain>.jfrog.io`.
  </Accordion>
  
  <Accordion title="Fly Web">
    The browser-based interface for managing Git connections, artifacts, releases, and team settings.
  </Accordion>
</AccordionGroup>

## G

<AccordionGroup>
  <Accordion title="Git Traceability">
    The automatic linking of every artifact to its source Git commit, branch, and repository.
  </Accordion>
</AccordionGroup>

## L

<AccordionGroup>
  <Accordion title="Local Repository">
    A repository in Fly that stores artifacts published by your team.
  </Accordion>
</AccordionGroup>

## M

<AccordionGroup>
  <Accordion title="MCP (Model Context Protocol)">
    Protocol that enables AI assistants to interact with Fly through the Desktop App.
  </Accordion>
</AccordionGroup>

## O

<AccordionGroup>
  <Accordion title="OIDC (OpenID Connect)">
    Authentication protocol that allows CI/CD systems to authenticate with Fly without storing long-lived tokens.
  </Accordion>
</AccordionGroup>

## P

<AccordionGroup>
  <Accordion title="Package Manager">
    Tools like npm, pip, Maven, or Docker that manage software dependencies and artifacts.
  </Accordion>
</AccordionGroup>

## R

<AccordionGroup>
  <Accordion title="Release">
    A versioned collection of artifacts associated with a Git tag, including release notes and changelog.
  </Accordion>
  
  <Accordion title="Remote Repository">
    A repository that proxies and caches packages from external sources like npmjs.org or Docker Hub.
  </Accordion>
</AccordionGroup>

## S

<AccordionGroup>
  <Accordion title="Semantic Search">
    The ability to find artifacts using natural language descriptions rather than exact names.
  </Accordion>
  
  <Accordion title="Semantic Versioning (SemVer)">
    Version numbering scheme: MAJOR.MINOR.PATCH (e.g., 2.1.0).
  </Accordion>
</AccordionGroup>

## V

<AccordionGroup>
  <Accordion title="Virtual Repository">
    A repository that aggregates multiple local and remote repositories into a single access point.
  </Accordion>
</AccordionGroup>

## W

<AccordionGroup>
  <Accordion title="Workflow">
    A CI/CD pipeline that interacts with Fly, typically for building and publishing artifacts.
  </Accordion>
</AccordionGroup>
